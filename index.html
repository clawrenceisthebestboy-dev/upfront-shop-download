How to verify the hash before running it

Open Command Prompt in the folder you downloaded to:

certutil -hashfile UpFrontShopSetup-1.4.0.exe SHA256

The long hex string it prints must exactly match the one above. If it doesn't, do not run the installer — something tampered with the download. Re-download; if it still doesn't match, call Josh.

Install

  1. Click the Download installer button above.
  2. (Optional but recommended) Verify the SHA-256 — see the expandable block.
  3. Double-click UpFrontShopSetup-1.4.0.exe.
  4. Click through the installer. Windows SmartScreen will warn — click More info → Run anyway the first time (we're signing it with a real cert in a future release).
  5. Start menu → Up Front Shop. Open Settings → Printer and pick the shop's WiFi printer. Open Settings → Shop Info and confirm phone/address/footer. You're done.
First-time only: the app creates its database at %APPDATA%\UpFrontShop\shop.db. Back that whole folder up to a USB stick weekly — that's your ledger.

After the first install, updates are automatic

When a new release is published here, the app picks it up the next time it launches (or right away from File → Check for Updates…). The updater will:

  1. Fetch the signed manifest from this server.
  2. Verify the Ed25519 signature against a public key baked into your installed copy. If the signature is bad — even if the download succeeded — the update is refused.
  3. Download the installer, verify its SHA-256 matches the manifest.
  4. Ask the tech to confirm, then run the installer silently. The app closes and re-opens on the new version.

If any of those steps fail, the update is thrown out and the app keeps running on the old version. You'll never get an unsigned or tampered installer.
